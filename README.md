ตั้งค่าตัวตนผู้ใช้งาน (Identity) ก่อนเริ่มบันทึกงาน:

#### สร้าง Repository ใหม่ของตนเองบน GitHub โดยตั้งชื่อตามรูปแบบ: it464-Test02-Docker-Container-2-[username ของนักศึกษา]

# 🏗️ 1. โจทย์สถานการณ์
ให้นักศึกษาติดตั้งระบบติดตามสถานะโครงงาน โดยเรียบเรียง 4 บริการ (Services) ให้ทำงานร่วมกันภายใต้เงื่อนไขความปลอดภัยและพอร์ตที่ได้รับมอบหมาย

    └── script-data.sql     # ไฟล์สำหรับเตรียมข้อมูลเริ่มต้น

# ⚙️ 2. ข้อกำหนดทางเทคนิค (20 คะแนน)
### 2.1 การตั้งค่าพอร์ต (Port Mapping) *สำคัญ
#### ให้นักศึกษาคำนวณพอร์ตของตนเองตามสูตรดังนี้ (เพื่อไม่ให้ชนกับผู้อื่น):
    Web Server (Nginx): 10000 + (เลขที่ * 100) + 1    เช่น เลขที่ 45 ใช้เลข 14501
    phpMyAdmin: 10000 + (เลขที่ * 100) + 2  เช่น เลขที่ 45 ใช้เลข 14502
### 2.2 บริการและเครือข่าย (Services & Networks)
#### Naming: ทุกคอนเทนเนอร์ต้องตั้งชื่อโดยมี Prefix เป็น Username ของนักศึกษา (เช่น ekachaikanya-db)
    Network Isolation: แบ่งเป็น 2 วง:
        frontend-zone: สำหรับ Web Server และ phpMyAdmin ติดต่อภายนอก
        backend-zone: วงปิดสำหรับให้ PHP ติดต่อกับ Database เท่านั้น
    Storage: ใช้ Named Volume ชื่อ db-storage สำหรับเก็บข้อมูล MySQL ถาวร

# 📝 3. โครงสร้างไฟล์สำหรับเติมคำตอบ (Skeletons)
### 📄 ตำแหน่งไฟล์
ให้นักศึกษาย้ายตำแหน่งไฟล์ให้ถูกต้องเอง โดยดูจากโครงสร้างไฟล์และไฟล์ compose.yaml ประกอบ

### 📄 ไฟล์ README.md
ให้นักศึกษา เพิ่มข้อความชื่อ-นามสกุล(อังกฤษ) รหัสนักศึกษา เลขที่ กลุ่มเรียน ขึ้นไปกรอกด้านบน

### 📄 ไฟล์ script-data.sql
ให้นักศึกษานำไปใช้ใน db-manangement

### 📄 ไฟล์ db_pass.txt
ให้นักศึกษาตั้งรหัสผ่านเอง

### 📄 ไฟล์ index.php
ให้นักศึกษาเพิ่มข้อความลงไฟล์เพื่อให้เชื่อมกับฐานข้อมูล โดยดูจากไฟล์ compose.yaml

### 📄 ไฟล์ .env
ให้นักศึกษาเติมข้อมูลของตนเองให้ครบถ้วน:  ตามตัวแปรที่มีในไฟล์ compose.yaml

MY_USERNAME=_________

HTTP_PORT=_________

PMA_PORT=_________

DB_NAME=_________

DB_USER=________
